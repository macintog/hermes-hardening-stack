## Highest-priority gaps found

1. Browser/web/document/Youtube extraction tool results can place hostile remote text directly into tool-result context. Add a shared context-safety wrapper for tool results or targeted wrappers for browser/extraction tools.
2. Skills hub remote fetch/install paths are protected by `skills_guard.py` after content is available, but download policy is not centralized. Add safe HTTP / GitHub API caps and redirect policy where feasible.
3. @ file/git/url reference searches did not find a dedicated parser in `agent tools cron gateway .`; equivalent risk is present through `read_file`, browser/web extraction, git/terminal, and context-file loading. If a parser exists outside those paths or is added later, enumerate remote fetch helpers, local file read limits, secret redaction, and promotion role.
4. Side-effecting tools (`terminal`, `execute_code`, file writes/deletes, send_message, cron, memory writes, skill/plugin install) need a policy boundary preventing untrusted downloaded/extracted instructions from authorizing actions.
