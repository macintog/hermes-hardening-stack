## Highest-priority gaps found

1. Platform-native extracted text/transcripts that bypass model/tool-result promotion still need targeted wrappers, especially gateway STT/OCR/document paths.
2. Skill supporting-file path hints and setup notes should continue to be reviewed separately from fenced skill body content.
3. @ file/git/url reference searches did not find a dedicated parser in `agent tools cron gateway .`; equivalent risk is present through `read_file`, browser/web extraction, git/terminal, and context-file loading. If a parser exists outside those paths or is added later, enumerate remote fetch helpers, local file read limits, secret redaction, and promotion role.
4. Browser network fetch behavior remains outside `safe_http` because browser operation is intentional and external-service mediated; the hardened boundary is model-visible browser/web extraction output, not browser navigation itself.
